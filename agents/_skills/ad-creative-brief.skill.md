# Ad Creative Brief Skill

Comprehensive knowledge for writing professional creative briefs for UGC video ads, static image ads, and multimedia campaigns.

## Brief Principles

- Every brief is a source of truth for the creative team
- Briefs must be actionable - creators and editors should be able to produce from the brief alone
- Always include strategic reasoning ("Why It Works") - never create without rationale
- Include 3 hook variants minimum for video briefs
- Include 3 headline variants minimum for static briefs

## Video Brief Structure

### Required Header Information

Every video brief must begin with:

| Field | Description |
|-------|-------------|
| **Concept Name** | Catchy, memorable name for this creative |
| **Client** | Client name |
| **Product Link** | Direct link to the product being advertised |
| **Target Audience** | Specific audience (age, mindset, funnel stage) |
| **Language** | Script language (German, English, etc.) |
| **Format** | Primary format + additional cuts (e.g., "Story 9:16 + 4:5 + 1:1") |
| **Duration** | Target length (e.g., "30 seconds") |
| **Platform** | Meta, TikTok, YouTube, etc. |

### "Why It Works" Section (Mandatory)

Always include strategic reasoning. Choose from these categories:

**Performance-Based:**
- "Iteration of [AD NAME] (spent X, Y.Z ROAS) - testing [CHANGE]"
- "Structure from top 5 performers: [PATTERN]"
- "Hook style from winning ad [AD ID] - applying to new angle"

**Research-Based:**
- "Based on customer review insight: '[QUOTE]'"
- "From competitor analysis: [COMPETITOR] format achieves [METRIC]"

**Hypothesis-Based:**
- "Testing hypothesis: [IF X THEN Y]"
- "A/B test of [VARIABLE]: Version A = [X], this brief = [Y]"

### Script Table Format (3 Columns)

| SCRIPT | VISUAL GUIDELINE | TEXT ON SCREEN + EDITOR'S NOTES |
|--------|------------------|----------------------------------|
| Spoken word | What the creator does / what we see | Text overlays, sounds, transitions, effects |

**Row structure:**
- HOOK 1: Own row
- HOOK 2: Own row
- HOOK 3: Own row
- BODY sections: One row per logical section
- CTA: Own row

### Dial Settings Table

Include for every brief to guide creative direction:

| Dial | Value | Effect on Content |
|------|-------|-------------------|
| Authenticity | 0-1 | How "ad-like" vs "native" |
| Production Complexity | 0-1 | Simple UGC to cinematic |
| DR Intensity | 0-1 | Brand awareness to hard sell |
| Product Clarity | 0-1 | Lifestyle-first to product hero |
| Hook Pre-qualification | 0-1 | Universal to audience-specific |
| Funnel Stage | TOF/MOF/BOF | Awareness / Consideration / Conversion |

### Creator Brief Section

The creator brief is what goes to the UGC creator. It must be self-contained and clear.

**Required elements:**
- Audience context (describe who they are talking to, not persona names)
- Vibe description (energy level + what it is NOT)
- Full script table with timing:

| Scene | Time | What You Do | What You Say | Notes |
|-------|------|-------------|--------------|-------|

- B-Roll shot list:

| Shot | Description | Priority | Notes |
|------|-------------|----------|-------|

- Environment do's and don'ts
- Technical checklist

### Editor Brief Section

**Always written in English** regardless of the ad's language.

Required elements:
- Structure steps (numbered edit sequence)
- Script with text overlays table
- English translation table (if ad is in another language)
- Audio direction (music, sound effects)
- Subtitle style specifications
- Typography guidelines
- CTA card design notes
- QA checklist for the editor

## Static Ad Brief Structure

### Concept Overview

| Element | Details |
|---------|---------|
| Concept Name | Descriptive name |
| Angle | Problem/Solution, Identity/Lifestyle, Social Proof, etc. |
| Target Persona | Specific audience description |
| Awareness Level | Unaware / Problem-Aware / Solution-Aware / Brand-Aware |
| Core Insight | Key insight from research driving this concept |

### Standard Angles for Static Ads

| Angle | Best For | Example |
|-------|----------|---------|
| Problem/Solution | Addressing pain points | "Never Orange Again" |
| Identity/Lifestyle | Aspirational positioning | "My Weekly Glow Up" |
| Social Proof | Trust building | "10,000+ 5-Star Reviews" |
| Before/After | Transformation | "See the Difference" |
| Fear of Missing Out | Urgency/scarcity | "Limited Edition" |
| Authority/Expert | Credibility | "Dermatologist Approved" |
| Comparison | Competitive positioning | "Why We're Different" |
| UGC Style | Authenticity | "Real Results" |

### Awareness Level Guide

| Level | What They Know | Hook Approach |
|-------|----------------|---------------|
| Unaware | Don't know they have a problem | Educate, provoke curiosity |
| Problem-Aware | Know the problem, not the solution | Agitate pain, introduce solution |
| Solution-Aware | Know solutions exist, comparing | Differentiate, prove superiority |
| Brand-Aware | Know your brand, need convincing | Social proof, offers, urgency |

### Visual Design Section

Include:
- Creative direction description (vibe, aesthetic)
- Photography/design style table (models, setting, poses, editing, energy)
- Layout description with ASCII mockup
- Visual elements specification (background, lighting, product placement, text placement, logo)

### Copy Elements

- 3 headline variants (A recommended, B and C alternatives)
- Supporting copy for each headline
- Social proof element (customer quote with name)
- 3 CTA options (primary + 2 alternatives)

### Design Specifications

| Element | Specification |
|---------|---------------|
| Dimensions | 1080x1080 / 1080x1350 / 1080x1920 |
| Color Palette | Brand hex codes |
| Typography | Font names and weights |
| CTA Button | Color, shape, style |

## UGC Creator Guidelines

### Before Shooting
- Read the ENTIRE brief before filming
- Note all required hooks (usually 3)
- Check the B-roll shot list
- Set up clean, uncluttered background
- Good lighting (natural preferred or ring light)
- Quiet location (no background noise)
- Phone/camera at eye level for talking head

### Technical Requirements
- Resolution: 1080p minimum, 4K preferred
- Orientation: Vertical (9:16) unless specified otherwise
- Frame Rate: 30fps standard
- Audio: Clear voice audio is CRITICAL - test before full shoot

### Delivery Style
- Natural, conversational tone - like talking to a friend
- Do not read from a script - memorize key points, deliver naturally
- Eye contact with the camera lens, not the screen
- Pacing: Slightly faster than normal conversation
- Film each hook 2-3 times with different energy/delivery
- Film each body section at least 2 times

### Common Mistakes to Avoid
- Reading from a script off-camera (viewers can tell)
- Robotic delivery
- Poor audio quality
- Shaky footage
- Forgetting B-roll shots
- Only filming each line once

### File Organization
Name files clearly: `hook1_take1.mp4`, `body_intro_take1.mp4`, `broll_product_closeup.mp4`

## Brief Checklist

### Video Brief
- [ ] "Why It Works" section with strategic reasoning
- [ ] Client website and product link included
- [ ] Target audience is specific (not just "broad")
- [ ] Language specified
- [ ] At least 3 hook variants
- [ ] Editor notes are clear and actionable
- [ ] Technical specs cover audio, music, caption requirements
- [ ] Format includes all aspect ratio cuts needed
- [ ] Dial settings table included
- [ ] B-roll shot list included

### Static Brief
- [ ] "Why It Works" section with strategic reasoning
- [ ] Client website and product link included
- [ ] Language specified
- [ ] Target persona with clear hook approach
- [ ] Awareness level matches the angle
- [ ] At least 3 headline variants
- [ ] Visual layout clearly described (ASCII mockup)
- [ ] Design specifications complete (dimensions, colors, fonts)
- [ ] Compliance check completed
- [ ] Shot list included (if photoshoot required)

## Language Rules for Briefs

- **Ad copy:** In the target language, using transcreation (not literal translation)
- **Editor instructions:** ALWAYS in English (editors are English-speaking)
- **Creator instructions:** In the language the creator speaks
- Non-English copy must sound native - read-aloud test is mandatory
